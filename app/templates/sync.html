How SCIM Sync Works
Import
Import from Netskope — Reads all users and groups from your Netskope tenant's SCIM API. Creates missing records locally and links existing ones by setting their scim_id.
Push
Push to Netskope — Sends local users and groups that have no scim_id yet to Netskope's SCIM API. Matches by email/name first before creating new records.
Server
SCIM Server — Exposes /scim/v2/Users and /scim/v2/Groups so any SCIM-compatible IdP (Okta, Azure AD) can provision directly into this server, which then syncs to Netskope.
